    # 1、介绍下 BFC、IFC、GFC 和 FFC 及其应用

    Formatting Contexts：格式化上下文，指页面中一个渲染区域，拥有一套渲染规则，它决定了其子元素如何定位，以及与其他元素的相互关系和作用。

    其实就是通过设置不同的displa值，让那个区域按照不同的规则渲染：

    • BFC：块级格式化上下文 =》 盒子垂直放置, 垂直间距决定于最大的margin属性 =》 display: inline-block;
    • IFC: 内联格式化上下文 =》 盒子水平放置，水平方向不会发生margin重叠，但垂直方向margin、padding无效 =》 display: inline;
    • GFC: 网格布局格式化上下文 =》 网格布局 =》 display: grid;
    • FFC: 自适应格式化上下文 =》 flex 布局 =》 display: flex/inline-flex;

    # 2、css 重绘和重排有什么区别？哪些css属性会导致重绘和重排？

    涉及到浏览器渲染

    重绘是当元素外观改变时引起浏览器重新绘制，重排是当元素几何信息和位置改变时引起的重新排版。

    引起重绘的属性：color、background、box-shadow等

    引起重排的属性：width、height、position、float等

    损耗：重排（layout） > 重绘（paint）

    解决：避免一次次改，最好批量改，动画最好做定位，不要影响整体布局，引发重排等。

    # 7、::before和:after中单冒号和双冒号的区别是什么，这两个伪元素有什么作用？

    单冒号是伪类，双冒号是伪元素。为了兼容，部分伪元素支持单冒号，如 :before 和 :after。

    ::before: 在某元素前插入内容, ::after: 在某元素后插入内容。

    常用场景：标题前 加 icon

    # 8、请描述 margin 边界叠加是什么及解决方案

    另一种问法：margin 坍塌是什么？怎么解决？

    原因：在 BFC 中,两个相邻块级盒子之间垂直方向上的外边距是会塌陷/叠加

    一般需要解决的是父子元素的 margin 叠加。解决方法：

    • 父 Div 设置 overflow:hidden;
    • 父 Div 设置 padding: 1px;
    • 父 Div 设置 border: 1px solid transparent;
    • 父 Div 设置 position: absolute;
    • 子 Div 设置 float:left;
    • 子 Div 设置 display: inline-block;

    # 9、什么是FOUC？你是如何避免FOUC的？

    FOUC: Flash of Unstyled Content（文档样式短暂失效），这种现象也称为浏览器样式闪烁。

    原因可能为：

    1. 使用import方法导入样式表
    2. 将样式表放在页面底部
    3. 有几个样式表，放在html结构的不同位置

    原理：

    当样式表晚于结构性html 加载，当加载到此样式表时，页面将停止之前的渲染。 此样式表被下载和解析后，将重新渲染页面，也就出现了短暂的花屏现象。

    解决方法：

    使用link标签将样式表放在文档head中

    # 11、rgba()和opacity这两个的透明效果有什么区别？

    opacity 作用于元素，以及元素内的所有内容的透明度，包括子元素。

    rgba() 只作用于元素的颜色或其背景色，并且设置 rgba 透明的元素的子元素不会继承透明效果！

    # 12、当一个元素被设置为浮动或定位后，它的display值会变成什么？

    所以如果行内元素设置浮动后就可以设置宽高了。

    # 15、介绍下position属性，position: sticky是什么

    position 属性可以设置元素的定位。它有 fixed 、 absolute 、 relative 、 static（默认值）、 inherit 、sticky（css3新增）可选。

    • static： 是默认值，代表没有定位，处于正常文档流
    • inherit： 表示继承父元素 position 属性，但 IE8 前不兼容
    • sticky：粘性定位，当滚动元素距离父元素达到 sticky 设置的要求时(如top: 100px)，此时出现 fixed 效果。但有兼容性问题。

    sticky 可以用于设置吸顶效果。

    # 16. 介绍下都有什么盒子模型

    盒子模型（Box Model）就是把HTML页面中的元素看作是一个矩形的盒子，也就是一个盛装内容的容器。每个矩形都由元素的内容(content)、内边距（padding）、边框（border）和外边距（margin）组成。

    盒子模型有两种：

    • w3c 标准盒子模型。盒子的宽高不包含padding和border。可通过设置box-sizing:content-box;转换为标准模式
    • IE 怪异盒子模型。盒子的宽高包含了padding和border。可通过设置box-sizing:border-box;转换为怪异模式
