# 1. v-if 和 v-show 的区别

v-if 会控制dom是否渲染，如果是false的话会将整个dom移除。v-show 是类似于给标签添加了一个style="display:none",在初始化的时候都会加载。

v-if 比 v-show 损耗性能。

使用场景：当频繁切换的时候，可以使用v-show。

# 2. v-for 为什么要加上 key，有什么作用？v-for 为什么不能跟 v-if 一起使用

v-for 可以循环数组，也可以循环对象。key 是作为唯一性，让 vue 在进行 diff 算法的时候更准确的识别。

v-for 的优先级比 v-if 高，每次列表渲染都会进行一次判断，如果列表数据很多的话，性能损耗严重。

3.0版本中。v-if 的优先级高于 v-for

# 3. v-if、v-show、v-html 的原理是什么，它是如何封装的

v-if 会调用 addIfCondition 方法，生成 vnode 的时候会忽略对应节点，render 的时候就不会渲染；

v-show 会生成 vnode，render 的时候也会渲染成真实节点，只是在 render 过程中会在节点的属性中修改 show 属性值，也就是常说的 display；

v-html 会先移除节点下的所有节点，调用 html 方法，通过 addProp 添加innerHTML属性，归根结底还是设置innerHTML为v-html的值。

# 4. computed 是如何实现的？它和 watch 的区别

computed 会搜集并记录依赖。

初始化 =》 initComputed - 给每个属性添加watcher 并且执行 definedComputed =》 definedComputed - createComputedGetter =》createComputedGetter =》判断是否 dirty

computed 是计算属性，watch 是监听属性，两者功能不一样。computed 有缓存，如果data值不改变，则不会重新计算，这样能提高性能。

watch 可以监听数据变化，对于引用类型，可以配置deep: true进行深度监听，但引用类型是拿不到oldVal的值的，因为指向都是同一个内存对象。

# 5. 介绍下 vue 的事件，它是挂载在哪里的？

vue 的事件通过 @ 关键字，默认会传一个 event，这个 event 是原生的 event 对象，通过event.currentTarget知道，事件是绑定在当前元素上的。

react 的事件是合成事件。

vue 的 v-for 循环是没有做事件代理的，如果有需要要自己手动代理。

# 7. 父子组件生命周期钩子执行顺序是什么

从生命周期过程来看，每个组件都需要先解析生成vdom然后再挂载，基于这个顺序，每个子组件都是在父组件内，那么父组件在生成 vdom 的过程势必要先解析子组件，将子组件挂载后，再挂载父组件（爷孙组件也是一样），如果有多个子组件，则按子组件的排列顺序。也就是：

# 父子生命周期

父 beforeCreate =》 父 created =》 父 beforeMounted =》 子 beforeCreate =》 子 created =》 子 beforeMounted =》 子 mounted =》 父 mounted

# 父子组件通讯

  1. 通过props 和 $emit，数据是单向的。子组件不能修改 props 值
  2. 绑定自定义事件。通过 vue 实例自带的事件方法可以做到。 $emit 触发、 $on 绑定. $off 解除

# 父子更新顺序

父 beforeUpdate =》 子 beforeUpdate =》 子 updated =》 父 updated

# 8. 子组件为什么不可以修改父组件传递的 prop

props 是单向数据流，父级的 props 更新会向下流动到子组件，但反过来不行。这样会防止从子组件意外变更父级组件的状态，从而导致你的应用的数据流向难以理解。

注意：在子组件内直接修改基本类型的 props 值会直接报错，但如果修改引用类型则不会，因为引用类型传递过去的是一个指针，子组件改变值并没有改变指针，所以不会报错

# 11. 谈一谈 nextTick 原理

vue 是异步渲染（ dom 改变后不会立刻渲染），当渲染完之后就会触发 nextTick 函数，可以把这个函数看成是vue dom 重新渲染后的回调函数。（所以不是每次数据修改都会触发，vue 会等所有的 data 都修改完之后才会触发）

nextTick 不会每次修改都触发，会等前面所有的data都修改完之后，一次性触发。

使用场景：当需要dom更新完之后再调用的时候

# 24. vue-router 里的<router-link>标签和<a>标签有什么区别

router-link 进行跳转不会跳转到新的页面，不会重新渲染，它会选择路由所指的组件进行渲染。在history 和 hash模式下表现一致

a标签跳转，页面就会重新加载，相当于打开一个新的页面

如果 router-link 中包含 a 标签，a 标签会取代 router-link 实现跳转

# 25. 路由模式：hash、history 模式的区别

路径不同：hash 会带有 “#” 号

实现原理：都是通过监听 url 上的变化，然后对浏览器的访问历史进行操作，只是调用Api有所不同。

vue-router 支持3种模式，默认使用hash模式。如果浏览器不支持 history 模式，则自动使用hash模式。

  • hash模式：调用window.locatio.replace()、window.location.href() 等api。（hash值得改变是不会刷新页面的）。

  • history模式：调用 HTML5 的history 有关API。window.history.replaceState()、window.history.pushState()等

  • abstract模式：非浏览器环境（如在node里使用）

# 27. 为什么 Vuex 的 mutation 和 Redux 的 reducer 中不能做异步操作？

因为 mutation 直接修改 state，如果 mutation 做异步处理，有多个异步操作的情况下，就无法判断哪一个才是最新值，造成数据混乱。

# 29. 聊聊 Redux 和 Vuex 的设计思想

它们提供了一个状态管理库，采用集中式存储管理应用的所有组件的状态。

在vue初始化的时候，会并将数据存放到全局的store，再将store挂载到每个vue实例组件中。

其他组件间共享数据的方法：

  • 父子组件间传值：$emit、props
  • 依赖注入(深层传值)：provide/inject
  • 直接获取组件，进行传值：ref、$parent/$child
