# 1.前端为何要进行构建和打包？

开发性题目，可以从几个方面结合项目拓展

  • 代码层面：
    • 体积更小（Tree-Shaking、压缩、合并），加载更快
    • 编译高级语言或语法（TS、ES6+，模块化，scss）
    • 兼容性和错误提检查（polyfill、postcss、eslint）
  • 研发流程
    • 统一、高效的开发环境
    • 统一的构建流程和产出标准
    • 集成公司构建规范（提测、上线等）

# 2.module、chunk、bundle分别什么意思，有何区别？

  • module：各个源码文件，webpack中一切皆模块
  • chunk：多模块合并成的，如 entry、import()、splitChunk
  • bundle：最终输出的文件

# 3.loader 和 plugin 的区别？

  • loader： 模块转换器，如 less -> css
  • plugin： 拓展插件，如 HTMLWebpackPlugin

# 4.常见 loader 和 plugin 有哪些？

  • 常见 loader：babel-loader、style-loader、css-loader、file-loader、url-loader、vue-loader
  • 常见 plugin：webpack-merge、html-webpack-plugin、mini-css-extract-plugin
  • awesome-webpack

# 5.babel 和 webpack 的区别？

  • babel：JS 新语法编译工具，不关心模块化
  • webpack：打包构建工具，是多个loader、plugin的集合

# 6.如何产出一个lib？

使用DLLPlugin

// 开发环境 - webpack.dll.js
module.exports = {
    entry: {
        react: ['react', 'react-dom']
    },
    output: {
        filename: '[name].dll.js',
        path: distPath,
        library: '_dll_[name]' // 第三方库名，加上 _dll_ 避免全局变量冲突
    }
    plugins: [
        new webpack.DllPlugin({
            name: '_dll_[name]',
            // 描述动态链接库的 manifest.json 文件输出时的文件名称
            path: path.join(distPath, '[name].manifest.json')
        })
    ]
}

# 7.babel-runtime 和 babel-polyfill的区别

  • babel-polyfill 会污染全局（corejs + regenerator）
  • babel-runtime不会污染全局
  • 产出第三方 lib 要用 babel-runtime

# 8.webpack 如何实现懒加载？

import()，可结合vue 、React 的异步组件，还有对应router异步加载路由

# 9.为什么 Proxy 不能被 polyfill？

  • Class 可以用 function 模拟
  • Promise 可以用 callback 模拟
  • Proxy 功能无法用 Object.defineProperty实现，无法模拟

# 10.webpack 常见性能优化手段

  • 优化打包构建速度 - 开发体验和效率
    • 优化babel-loader
    • IgnorePlugin
    • noParse
    • happyPack：开启多进程打包，提高构建速度
    • ParalleUglifyPlugin：多进程压缩 JS（webpack 内置 Uglify 工具压缩 JS）
    • 自动刷新
    • 热更新
    • DllPlugin
  • 优化产出代码 - 产品性能
    • 小图片，base64编码
    • bundle 加 hash
    • 懒加载
    • 提取公共代码
    • IgnorePlugin
    • 使用 CDN 加速：设置 publicPath（图片需要在loader中设置publicPath）
    • 使用 production：设置 mode: production
    • 使用 Scope Hosting(作用域提升)：代码合并（必须使用ES6 module 的文件）
