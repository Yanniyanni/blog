# 1. 介绍下浏览器跨域

协议://地址:端口 三者都相同，才算同源，否则报跨域错误。

如果非同源，以下的行为受到限制：

  • Cookie、LocalStorage 和 IndexDB 无法读取
  • DOM 无法获得（iframe无法通信）
  • AJAX 请求不能发送

# 9. 403、301、302是什么？

  • 403：Invalid CORS。CORS 验证失败，包括origin、header、methods的不匹配都有可能抛出403。
  • 301：永久移动。请求的资源已被永久的移动到新URI，返回信息会包括新的URI，浏览器会自动定向到新URI。今后任何新的请求都应使用新的URI代替
  • 302：临时移动。与301类似。但资源只是临时被移动。客户端应继续使用原有URI

# 10. 缓存相关的HTTP请求头

请求头：

  • Cache-Control
  • expires(兼容旧版)
  • IF-Modified-Since:【date】 + Last-Modified
  • If-None-Match:【ETag】 + ETag

缓存数据返回状态可能是：

  • 304 Not Modified：服务器判断可以读取缓存，客户端直接读缓存
  • 200 OK (from memory cache)：从内存缓存中读取
  • 200 OK (from disk cache) ：从磁盘中读取
