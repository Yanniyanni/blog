# 1、var、let 和 const 区别的实现原理是什么? 介绍下什么暂时性死区？

三者都是用于声明变量。let 、const 是 es6 新增的。var 可以重复声明，并且会进行变量提升。let 和 const 则不会，如果未声明使用，就是报错，这在语法上就构成了“暂时性死区”。

在暂时性死区内使用变量会出现报错，如typeof x （如果是 var 声明，则会输出undefined）。

我们保持良好变量声明习惯，尽量避免触发。

# 2、变量提升和函数提升是什么？

变量提升和函数提升是在 js 执行的时候会将他们的声明提升到最顶部，从而使得在还没声明的时候能使用，此时值为 undefined。

注意，只有声明被提升，初始化不会被提升。

函数提升优先级高于变量提升。

《变量》

# 3、var的变量提升底层原理是什么？

关于 js 的执行过程。js 会先进行词法分析，再去执行，在分析阶段，就会将这些声明提升上去。

let、const 不会有变量提升。

《变量》

# 4、全局作用域中，用 const 和 let 声明的变量不在 window 上，那到底在哪里？如何去获取？

使用 var 声明的变量会挂载在 window 上，const 、 let 不在window上，它们存在当前作用域中。通过变量名就能获取到。

var x = 100;
const y = 99;

console.log(window.x); // 100
console.log(window.y); // undefined
console.log(y); // 99

# 5、什么是防抖和节流？有什么区别？如何实现？

防抖：指的是无论某个动作被连续触发多少次，直到这个连续动作停止后，才会被当作一次来执行。比如页面滚动。

节流：指的是无论某个动作被连续触发多少次，在定义的一段时间之内，它仅能够触发一次。比如resize和scroll时间频繁触发的操作，如果都接受了处理，可能会影响性能，需要进行节流控制。

实现：通过设置某个时间点，通过 setTimeout 延迟执行。

# 12、堆和栈的区别？它们具体怎么存储（引用类型的存储）

堆：一块内存空间，无序存储，存储引用类型（对象）

# 16、定时器为什么是不精确的？

因为 js 是单线程，只有一个线程在跑，所有的任务都需要按顺序执行，js 的执行模型是基于事件循环的。setTimeout 属于宏任务，事件循环机制中，先同步后异步，先宏任务再微任务。 即使是设置了某个时间点执行，如果当前执行栈所花费的时间大于定时器时间，那么定时器的回调在宏任务里，来不及调用，就会出现误差

# 17、setTimeout倒计时为什么会出现误差？如何校准？

原因：因为 js 是单线程，只有一个线程在跑，所有的任务都需要按顺序执行，js 的执行模型是基于事件循环的。setTimeout 属于宏任务，事件循环机制中，先同步后异步，先宏任务再微任务。 即使是设置了某个时间点执行，如果当前执行栈所花费的时间大于定时器时间，那么定时器的回调在宏任务里，来不及调用，就会出现误差

校准：setTimeout 的间隔时间不要写死，每次setTimeout的时候计算一下实际的间隔时间，从而修正下一次的间隔时间。

//继续线程占用
setInterval(function(){ 
     var j = 0; 
     while(j++ < 100000000); 
}, 0); 
 
//倒计时
var  interval = 1000,
       ms = 50000,  //从服务器和活动开始时间计算出的时间差，这里测试用50000ms
       count = 0,
       startTime = new Date().getTime();
if( ms >= 0) {
    var timeCounter = setTimeout(countDownStart,interval);                  
}
 
function countDownStart(){
    count++;
    var offset = new Date().getTime() - (startTime + count * interval); // 误差时间
    var nextTime = interval - offset; // 下次执行时间
    var daytohour = 0; 
    if (nextTime < 0) { 
        nextTime = 0 
    };
    ms -= interval;
    console.log("误差：" + offset + "ms，下一次执行：" + nextTime + "ms后，离活动开始还有：" + ms + "ms");
    if (ms < 0) {
        clearTimeout(timeCounter);
    } else {
        timeCounter = setTimeout(countDownStart,nextTime);
    }
 }

# 18、介绍宏任务和微任务

宏任务和微任务涉及到的是事件循环有关。事件循环又可以分为同步、异步任务。在这之上又可以根据以下区分为宏任务和微任务。

在浏览器的事件循环模型中：

  • 宏任务：同步代码、setTimeout、setInterval等

  • 微任务：Promise 后的then语句、process.nextTick等

执行顺序：先同步在异步，先宏任务再微任务。

浏览器的事件循环模型跟 node 的事件循环模型有差异。

# 34、什么是事件冒泡和事件捕获以及事件委托

涉及 js 的事件机制，事件流分三个阶段》事件捕获 - 目标事件 - 事件冒泡

事件冒泡：自下而上，从目标元素开始，往顶层元素传播。

事件捕获：自上而下，从顶层对象 document 发出一个事件流，随着DOM树的节点向目标元素节点流去。

事件冒泡和事件捕获都可以通过 event.stopPropagation() 阻止。

事件委托：利用事件冒泡原理，将事件委托给父级元素，通过一个事件管理某一类所有事件。事件委托是提升性能的方法之一。

《事件》

# 44、如何用 promise 包装 ajax

把 ajax return 返回一个 promise

function myAsyncFunction(url) {
    return new Promise((resolve, reject) => {
        const xhr = new XMLHttpRequest();
        xhr.open("GET", url);
        xhr.onload = () => resolve(xhr.responseText);
        xhr.onerror = () => reject(xhr.statusText);
        xhr.send();
    });
  };
