# 前端缓存介绍。

# html5 application cache应用缓存原理以及使用

HTML5 引入了应用程序缓存，这意味着 web 应用可进行缓存，并可在没有因特网连接时进行访问。

主要还是用在移动端（web 应用），例如几乎不会被修改的一些静态文件，公用的css，图片等

使用：html中添加manifest属性即可

<html manifest="demo.appcache">
1
2

demo.appcache是一个文本文件，里面包含三个方面

  • CACHE MANIFEST - 在此标题下列出的文件将在首次下载后进行缓存

  • NETWORK - 在此标题下列出的文件需要与服务器的连接，且不会被缓存

  • FALLBACK - 在此标题下列出的文件规定当页面无法访问时的回退页面（比如 404 页面）

# 优势：
  • 离线浏览 - 用户可在应用离线时使用它们
  • 速度 - 已缓存资源加载得更快
  • 减少服务器负载 - 浏览器将只从服务器下载更新过或更改过的资源。
# 缺点：
  • 更新建立在manifest文件的更新
  • 更新是全局性的，无法实现当个文件更新
  • 修完文件，第一次不生效，获取的仍然是缓存数据的内容只有第二次刷新才会生效
