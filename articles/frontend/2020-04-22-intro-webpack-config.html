          《webpack 常用配置说明》

          2020-04-22 上次更新时间：8/4/2020, 8:24:44 PM 0

          # 文件规划

          不同的环境有不同的配置需求，需做好环境区分。可以根据需求新增其他配置文件。

          build 文件夹下

          • webpack.base.config.js （基础配置）
          • webpack.dev.config.js （在基础配置上新增 dev 有关配置）
          • webpack.prod.config.js （在基础配置上新增 prod 有关配置）

          config 文件夹下

          • dev.js （dev 配置文件）
          • prod.js （prod 配置文件）
          • index （config 入口文件）

          # 常用 loader

          vue 处理 vue-loader

          style-loader 和 vue-style-loader的区别。为何使用 vue-style-loader 替代style-loader

          js 处理 babel-loader （cache-loader） js 校验 eslint (eslint-loader)

          css 处理 vue-style-loader、css-loader、(node-sass、sass-loader)、 postcss-loader ?? 加载 node_moduels 中的 CSS 之前还要使用 postcss-loader 再统一处理一遍，以确保所有进入生产环境的 CSS 都经过了相应的浏览器兼容性处理。 postcss-px-to-viewport

          sass-resources-loader

          css 处理工具 postcss、autoprefixer css 压缩 mini-css-extract-plugin

          资源文件文件 url-loader url-loader 功能类似于 file-loader，但是在文件大小（单位 byte）低于指定的限制时，可以返回一个 DataURL。 但是可以配置的没有file-loader丰富。

          file-loader 将webpack将所需的对象作为文件发送，并返回其公共URL.

          自定义 loader

          terser-webpack-plugin 允许提供一个或多个定制过的 TerserPlugin 实例，覆盖默认压缩工具(minimizer)。

          mini-css-extract-plugin css 压缩

          webpack-merge 合并配置

          cross-env 环境处理

          rimraf 删除操作

          html-webpack-plugin 处理html文件

          markdownit、hightlight.js 处理 markdown 文件

          progress-bar-webpack-plugin

          copy-webpack-plugin
