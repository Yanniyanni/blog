# 1、如何理解html语义化？有什么优缺点吗？

语义化就是用合理、正确的标签来展示内容，例如使用 h 标签来定义标题。html5 中新增了很多语义化标签，如<header>、<nav>、<section>。

优点：便于用户阅读、有利于seo、便于维护、方便机器解析

缺点：html5 标签有兼容性

延伸问题：都有哪些标签？

  • 块级标签：div、h标签、p标签、header、footer、nav、section等
  • 行内标签: span、a标签、i标签等

延伸问题：标签如何转换？

设置display:block/inline-block/inline;

延伸问题：display都有哪些值？

none、block、inline、inline-block、table、flex等。

延伸问题：flex 布局有关等

# 2、介绍下表单提交，和formData有什么关系？formData和原生的ajax有什么区别？

表单提交是通过设置<form>有关属性 action、method 等，通过绑定 submit 按钮触发浏览器行为，从而发送数据。是浏览器行为。

formData 是通过 FormData() 构造函数创建一个新的 FormData 对象，从而获得表单值。是用户自发行为

ajax 是通过 XmlHttpRequest 对象向服务器发异步请求，是用户自发行为

可能延伸到enctype、content-type 和 异步知识

《参考》

# 3、关于<form>标签的enctype属性你有哪些了解？

通过设置 enctype 属性，可以指定表单提交的 content-type 。可选值有application/x-www-form-urlencoded（默认）、multipart/form-data、text/plain

data- 用于存储自定义属性。原生 Js 可以通过 setAttribute() 和 getAttribute() 去设置和存储。

# 5、如何实现应用缓存？说说你对manifest的理解

html5 引入了应用缓存（又称离线缓存），可以让 web 应用在断网的时候访问。通过给html设置 manifest 属性设置文档缓存的位置。

优点：

  • 离线浏览 - 用户可以在离线时使用应用程序
  • 快速 - 缓存的资源可以更快地加载
  • 减少服务器加载 - 浏览器只从服务器上下载已更新/已更改的资源

缺点：缓存大量的数据会占用客户端存储空间，每次更新需要替换 manifest 文件并刷新缓存

使用：

index.html

<!DOCTYPE html>
<html manifest="/index.appcache">
    <head>
        <link rel="stylesheet" type="text/css" href="https://ss1.bdstatic.com/5eN1bjq8AAUYm2zgoY3K/r/www/cache/static/protocol/https/soutu/css/soutu_new2_ae491b7.css" />
        <script src="https://dss0.bdstatic.com/5aV1bjqh_Q23odCf/static/superman/js/lib/jquery-1-edb203c114.10.2.js"></script>
    </head>
    <body>
        <img src="https://dss0.bdstatic.com/5aV1bjqh_Q23odCf/static/superman/img/topnav/baiduyun@2x-e0be79e69e.png">
    </body>
</html>

# 6、HTML 文档中 DTD 的意义和作用

DTD 文档类型定义，一组机器可读规则。在 HTML 是对文档的声明，还会影响浏览器的渲染模式。

浏览器渲染模式：标准模式（严格模式）、混杂模式、准标准模式

html5 中不需要对 DTD 进行引用，只需写<!DOCTYPE html>。原因： html5 不基于 SGML，因此不需要对 DTD 进行引用，但仍需指定 doctype 来规范浏览器行为。
